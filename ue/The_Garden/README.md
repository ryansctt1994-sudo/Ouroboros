# The Garden — Engine & Toolchain Reference

> **Lock this version across all build machines before Week 1 Day 1.**  
> Any deviation produces "works on my machine" failures that are hard to diagnose.

---

## Engine Version

| Item | Value |
|------|-------|
| **Unreal Engine** | **5.4.4** (latest 5.4 hotfix as of 2026-03) |
| Install method | Epic Games Launcher — Library → Add version → 5.4.4 |
| Source build | Not required; Launcher binary is the shipping baseline |

If a critical engine bug forces an upgrade, update this file and the
`EngineAssociation` field in `The_Garden.uproject` in the same commit.
All three build machines must be updated before the next CI run.

## Required Toolchains per OS

### Windows
| Tool | Required Version | Notes |
|------|-----------------|-------|
| Visual Studio | 2022 (17.x latest) | Community edition is fine |
| MSVC workloads | "Game development with C++" | Includes Clang/LLVM for cross-compile |
| Windows SDK | 10.0.22621 or later | Installed via VS Installer |
| .NET SDK | 6.x (bundled with VS 2022) | For UnrealBuildTool |

Install path convention: `C:\Program Files\Epic Games\UE_5.4\`  
Set `UE_ROOT` runner variable to this path for CI.

### Linux
| Tool | Required Version | Notes |
|------|-----------------|-------|
| Clang | 16.x (Epic-provided toolchain) | Download from UE Linux prerequisites |
| UE Linux toolchain | v22 (for UE 5.4) | `UnrealToolchain-5.4-v22-Linux.tar.gz` |
| cmake | 3.22+ | Usually via distro package manager |
| libc++ / libstdc++ | Matched to clang 16 | |
| Ubuntu | 22.04 LTS recommended | Other distros work; 22.04 is most tested |

Install path convention: `/opt/UnrealEngine/5.4/`  
Set `UE_ROOT` runner variable to this path for CI.

Epic Linux toolchain setup:
```bash
# Download from: https://github.com/nicowillis/ue-linux-toolchains (or Epic's CDN)
mkdir -p /opt/ue-toolchain
tar -xzf UnrealToolchain-5.4-v22-Linux.tar.gz -C /opt/ue-toolchain
export LINUX_MULTIARCH_ROOT=/opt/ue-toolchain/v22_clang-16.0.6-centos7
```

### macOS
| Tool | Required Version | Notes |
|------|-----------------|-------|
| Xcode | 15.x (15.4 recommended) | From Mac App Store |
| macOS SDK | 14.x (bundled with Xcode 15) | |
| Command Line Tools | Bundled with Xcode | `xcode-select --install` if missing |
| Apple Developer ID | Required for notarisation | Signing is deferred; signing plan in `steam/README.md` |

Install path convention: `/Users/Shared/Epic Games/UE_5.4/`  
Set `UE_ROOT` runner variable to this path for CI.

## BuildInfo.json (generated at package time)

Every packaged build writes `Content/BuildInfo/BuildInfo.json`:

```json
{
  "game":     "The Garden",
  "version":  "0.1.0",
  "commit":   "abc1234",
  "branch":   "main",
  "config":   "Shipping",
  "platform": "Win64",
  "date":     "2026-03-15T12:00:00Z"
}
```

Generated by `scripts/build/generate-build-metadata.sh`.  
Loaded at runtime by `UTheGardenGameInstance::Init()` for log output and menu display.

## Steam Dev AppID

Development builds use **AppID 480** (Valve's public Spacewar test title).  
Place `steam_appid.txt` containing `480` next to the executable for non-Steam local runs.  
**Do not commit `steam_appid.txt`** — it is in `.gitignore`.

Real AppID: pending Steamworks partner application (required before Week 2).
